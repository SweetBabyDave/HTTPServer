Phase 0: Requirements Specification

I need to write a website for a fictional client that allows them to write their own blog. This website will have an about section featuring the client. It will also have CSS Tips as well. This program is aimed to generate a level of familiarity with HTML.

Good Solution

A good solution has all of the sections clearly labeling and links to each of them. The CSS and CSS- page look very similar, if not the same, with helpful information about CSS. There are pictures in the about section, and clearly labelled blog posts in the blog section. There are also no errors when ran through the validators.

Challenges

One challenge I see is creating all of the different styles for the CSS- page. It seems tedious and it might get messy with some of the errors seen in the validator. Making sure that I understand that information will be key when I am writing up the document.

Knowns

Phase 4: Testing and Debugging

I have ran my documents through the HTML and CSS validator a lot as I have written up the blog. The most common errors were:
  • There is a ending </p> without a <p> element in scope.
  • There are multiple uses of the #id when there should only be one.
For the first one, so long as I made sure that the p element was used a little more liberally at the beginning and end of multiple paragraphs, it helped to fix the error. The next one I couldn't figure out, but as long as I only used one instance of the id, the error was fixed. If you want to run the tests just go to:
  • HTML Validator
  • CSS Validator
